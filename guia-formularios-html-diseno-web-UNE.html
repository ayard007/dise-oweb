01

Fundamentos y Estructura Basica

Anatomia de un formulario y recoleccion de texto simple

El objetivo de este modulo es comprender la anatomia de un formulario HTML y como recolectar texto simple del usuario mediante campos de entrada accesibles.

La etiqueta contenedora: <form>

Todo formulario HTML inicia con la etiqueta <form>, que actua como contenedor de todos los campos de entrada. Sus dos atributos principales determinan que sucede con los datos capturados.

AtributoFuncionValores comunes
actionDefine a donde se envian los datos del formulario (una URL, un endpoint, un correo)"procesar.php", "mailto:correo@dominio.com", una URL de servicio externo
methodDefine como viajan los datos hacia el destinoGET o POST

method="GET"

Los datos viajan visibles en la URL (ej. ?nombre=Ana). Util para busquedas o filtros. No recomendado para informacion sensible, ya que queda registrado en el historial del navegador.

method="POST"

Los datos viajan ocultos en el cuerpo de la peticion, no en la URL. Es el metodo estandar para formularios de registro, contacto o cualquier envio de informacion personal.

form-base.html 
<form action="procesar-datos.php" method="POST">
  <!-- Aqui dentro viviran todos los campos del formulario -->
</form>

Cajas de texto: <input type="text">

El campo de texto es la entrada mas basica. Se utiliza para capturar cadenas cortas como nombres, apellidos o cualquier dato de una sola linea.

input-texto.html 
<input
  type="text"
  name="nombre"
  id="campo-nombre">
ⓘ
El atributo name es el identificador con el que viajara el dato hacia el servidor o servicio de destino. Sin name, el valor capturado no se envia.

Accesibilidad: la etiqueta <label>

La etiqueta <label> vincula un texto descriptivo a su campo de entrada correspondiente. Esta vinculacion no es solo visual: permite que lectores de pantalla identifiquen cada campo y que el usuario pueda hacer clic en el texto para activar el campo.

label-accesible.html 
<label for="campo-nombre">Nombre completo</label>
<input
  type="text"
  name="nombre"
  id="campo-nombre">
⚠
Regla de vinculacion: El valor del atributo for en <label> debe coincidir exactamente con el valor del atributo id en el <input> correspondiente. Sin esta coincidencia, la vinculacion de accesibilidad no funciona.

Textos extensos: <textarea>

A diferencia de <input type="text">, que es una etiqueta vacia de una sola linea, <textarea> es una etiqueta con apertura y cierre que permite multiples lineas de texto. Es ideal para comentarios, mensajes o descripciones extensas.

AtributoFuncion
rowsNumero de filas visibles (controla la altura)
colsNumero de columnas visibles (controla el ancho aproximado)
textarea.html 
<label for="campo-mensaje">Comentarios</label>
<textarea
  name="mensaje"
  id="campo-mensaje"
  rows="5"
  cols="40"></textarea>

Atributos de usabilidad: placeholder y value

Estos atributos mejoran la experiencia del usuario al interactuar con el campo, aunque cumplen funciones distintas.

placeholder

Texto de ayuda que aparece dentro del campo vacio y desaparece al escribir. No se envia como dato; es solo una guia visual.

value

Establece un valor por defecto que ya aparece escrito en el campo y que SI se envia si el usuario no lo modifica.

placeholder-value.html 
<input type="text" name="apellido"
       placeholder="Ej. Hernandez Lopez">

<input type="text" name="pais"
       value="Mexico">

Controles de Seleccion e Interaccion

De escribir a elegir: mejora de experiencia y estandarizacion de datos

Pasamos de capturar texto libre a ofrecer opciones predefinidas, lo que mejora la experiencia del usuario y estandariza la informacion recibida para su posterior analisis.

Opciones unicas: <input type="radio">

Los botones de radio permiten que el usuario elija una sola opcion entre varias. La clave tecnica es que todos los botones de un mismo grupo deben compartir el mismo valor en el atributo name; esto le indica al navegador que son mutuamente excluyentes.

radio-buttons.html 
<p>Selecciona tu turno:</p>

<input type="radio" name="turno"
       id="turno-matutino" value="matutino">
<label for="turno-matutino">Matutino</label>

<input type="radio" name="turno"
       id="turno-vespertino" value="vespertino">
<label for="turno-vespertino">Vespertino</label>
⚠
Observa que el atributo name es identico ("turno") en ambos botones, mientras que id es unico para cada uno. Esta es la diferencia critica entre agrupar y vincular accesibilidad.

Opciones multiples: <input type="checkbox">

Las casillas de verificacion permiten seleccionar varios elementos de manera simultanea. A diferencia del radio, cada checkbox funciona de forma independiente, incluso si comparten el mismo name (en cuyo caso se agrupan como un arreglo de valores).

checkboxes.html 
<p>Selecciona tus intereses extracurriculares:</p>

<input type="checkbox" name="intereses"
       id="int-deportes" value="deportes">
<label for="int-deportes">Deportes</label>

<input type="checkbox" name="intereses"
       id="int-arte" value="arte">
<label for="int-arte">Arte y cultura</label>

<input type="checkbox" name="intereses"
       id="int-voluntariado" value="voluntariado">
<label for="int-voluntariado">Voluntariado</label>

Menus desplegables: <select> y <option>

Cuando existen muchas opciones posibles (paises, estados, carreras universitarias), el menu desplegable es la solucion mas eficiente en espacio. <select> es el contenedor y cada <option> representa una alternativa.

select-option.html 
<label for="campo-carrera">Selecciona tu carrera</label>
<select name="carrera" id="campo-carrera">
  <option value="">-- Selecciona una opcion --</option>
  <option value="diseno-grafico">Diseno Grafico</option>
  <option value="pedagogia">Pedagogia</option>
  <option value="psicologia">Psicologia</option>
</select>
ⓘ
La primera opcion con value="" y un texto guia ("-- Selecciona una opcion --") es una buena practica de usabilidad: evita que el usuario envie el formulario sin haber elegido conscientemente una opcion.

Botones de accion

El formulario necesita un mecanismo para enviarse. Existen dos botones principales con comportamientos opuestos.

ElementoFuncionConsideracion
<button type="submit">Envia los datos del formulario al destino definido en actionEs el boton principal de toda interaccion
<input type="submit">Misma funcion, version mas antigua y limitada en estilosSe prefiere <button> por flexibilidad de diseno
<button type="reset">Limpia todos los campos del formulario a su estado inicialUsar con precaucion: el usuario puede perder informacion ya escrita sin posibilidad de recuperarla
botones-accion.html 
<button type="submit">Enviar registro</button>
<button type="reset">Limpiar formulario</button>
⚠
Por que usar reset con precaucion: Un usuario que llevaba varios minutos completando un formulario extenso puede perder todo su trabajo con un solo clic accidental. Si decides incluirlo, separalo visualmente del boton de envio y considera anadir confirmacion antes de borrar.

03

Validacion Nativa con HTML5

Asegurar datos correctos antes del envio, sin programar en otros lenguajes

HTML5 incorpora mecanismos de validacion automatica que el navegador ejecuta antes de permitir el envio del formulario, sin necesidad de escribir codigo en JavaScript u otros lenguajes.

Tipos de datos especificos

El atributo type de <input> no solo determina la apariencia del campo, tambien activa validacion automatica del navegador segun el formato esperado.

TipoValidacion automaticaEjemplo de uso
type="email"Exige formato usuario@dominioCorreo de contacto
type="number"Solo acepta valores numericosEdad, cantidad
type="date"Despliega selector de calendario, exige formato de fecha validoFecha de nacimiento
type="password"Oculta visualmente los caracteres ingresadosContrasenas
tipos-input.html 
<label for="campo-correo">Correo electronico</label>
<input type="email" name="correo" id="campo-correo">

<label for="campo-edad">Edad</label>
<input type="number" name="edad" id="campo-edad">

<label for="campo-fecha">Fecha de nacimiento</label>
<input type="date" name="fecha" id="campo-fecha">

<label for="campo-pass">Contrasena</label>
<input type="password" name="clave" id="campo-pass">

Campos obligatorios: required

El atributo required impide que el formulario se envie si el campo esta vacio. El navegador muestra automaticamente un mensaje de validacion senalando el campo pendiente.

required.html 
<input type="text" name="nombre" required>

Limites y rangos

Estos atributos restringen el rango de valores aceptables, ya sea numerico o en longitud de texto.

AtributoAplica aFuncion
mintype="number", type="date"Valor minimo permitido
maxtype="number", type="date"Valor maximo permitido
minlengthtype="text", type="password"Cantidad minima de caracteres
maxlengthtype="text", type="password", textareaCantidad maxima de caracteres
limites-rangos.html 
<label for="campo-edad2">Edad (entre 18 y 65 anios)</label>
<input type="number" name="edad" id="campo-edad2"
       min="18" max="65" required>

<label for="campo-matricula">Numero de matricula</label>
<input type="text" name="matricula" id="campo-matricula"
       minlength="6" maxlength="10" required>

Validacion avanzada: el atributo pattern

El atributo pattern permite definir una expresion regular (regex) que el valor del campo debe cumplir. Es util para forzar formatos especificos que los tipos predefinidos de HTML no cubren, como un numero de matricula con estructura particular.

pattern-regex.html 
<!-- Exige el formato: 2 letras + 6 numeros, ej. DG123456 -->
<label for="campo-mat2">Matricula institucional</label>
<input type="text" name="matricula" id="campo-mat2"
       pattern="[A-Z]{2}[0-9]{6}"
       title="Formato esperado: 2 letras mayusculas seguidas de 6 numeros, ej. DG123456"
       required>
ⓘ
El atributo title junto a pattern es fundamental: el navegador muestra ese texto como mensaje de ayuda cuando el formato ingresado no coincide con la expresion regular.

04

Procesamiento y Envio de Datos

El mundo real: donde el frontend se conecta con un servicio externo

Esta es la fase culminante del aprendizaje sobre formularios: conectar la estructura visual que ya dominas con servicios reales que almacenan la informacion capturada, ya sea en un correo electronico o en una hoja de calculo en linea.

Envio a correo electronico

El metodo clasico (limitado): mailto:

Es posible usar mailto: como valor del atributo action, lo cual intenta abrir el cliente de correo predeterminado del usuario con los datos precargados.

mailto-metodo.html 
<form action="mailto:contacto@une.edu.mx" method="POST"
      enctype="text/plain">
  <!-- Campos del formulario -->
</form>
⚠
Por que es limitado: Este metodo depende por completo de que el usuario tenga configurado un cliente de correo de escritorio (como Outlook) en su computadora. Si solo usa Gmail desde el navegador, el formulario fallara o se comportara de forma inconsistente. Por esta razon, mailto: no se considera una practica profesional actual.

El metodo moderno (recomendado): servicios de terceros

Servicios como Formspree o EmailJS ofrecen un punto de enlace (endpoint) propio: una URL a la que tu formulario envia los datos mediante POST, y el servicio se encarga de reenviarlos a tu correo de forma confiable, sin importar el cliente de correo del usuario.

formspree-metodo.html 
<!-- El endpoint se obtiene al crear una cuenta gratuita en formspree.io -->
<form action="https://formspree.io/f/tu-id-unico" method="POST">
  <label for="f-nombre">Nombre</label>
  <input type="text" name="nombre" id="f-nombre" required>

  <label for="f-correo">Correo</label>
  <input type="email" name="_replyto" id="f-correo" required>

  <button type="submit">Enviar</button>
</form>
1

Ingresa a formspree.io y registrate con tu correo institucional o personal.

2

4

Envio a hoja de calculo en linea (Google Sheets)

Concepto de Webhook

Un Webhook es una URL que "escucha" solicitudes y ejecuta una accion automatica al recibirlas. En este caso, configuraremos una URL que recibe los datos enviados por nuestro formulario HTML y los escribe automaticamente como una nueva fila en una hoja de Google Sheets.

El usuario llena y envia el formulario HTML
↓
El navegador realiza una peticion POST hacia la URL del Webhook
↓
Google Apps Script recibe la peticion y procesa los datos
↓
Se escribe una nueva fila en Google Sheets en tiempo real

Implementacion con Google Apps Script

Esta ruta ofrece control total y no depende de servicios de terceros pagados. Sigue estos pasos para crear tu propio Webhook.

1

Crea una hoja de calculo nueva en Google Sheets

Nombrala segun tu proyecto, por ejemplo "Registro Evento Escolar". En la primera fila escribe los encabezados de columna: Fecha, Nombre, Correo, Turno.

2

Abre el editor de Apps Script

En el menu de tu hoja, ve a Extensiones > Apps Script. Se abrira un editor de codigo en una pestania nueva.

3

Escribe el script que procesa el formulario

Borra el contenido por defecto y pega el codigo que recibira los datos POST (ver bloque de codigo abajo).

4

Implementa el script como aplicacion web

Clic en Implementar > Nueva implementacion. Selecciona tipo "Aplicacion web". En "Quien tiene acceso" elige "Cualquier usuario". Copia la URL que se genera.

5

Pega esa URL en el action de tu formulario HTML

Esa URL es ahora tu Webhook funcional.

Codigo.gs (Google Apps Script) 
function doPost(e) {
  var hoja = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  var datos = e.parameter;

  hoja.appendRow([
    new Date(),
    datos.nombre,
    datos.correo,
    datos.turno
  ]);

  return ContentService
    .createTextOutput(JSON.stringify({ "resultado": "exito" }))
    .setMimeType(ContentService.MimeType.JSON);
}
formulario-sheets.html 
<form action="https://script.google.com/macros/s/TU-ID-UNICO/exec"
      method="POST">

  <label for="s-nombre">Nombre completo</label>
  <input type="text" name="nombre" id="s-nombre" required>

  <label for="s-correo">Correo</label>
  <input type="email" name="correo" id="s-correo" required>

  <button type="submit">Registrarme</button>
</form>
ⓘ
Alternativa sin codigo: Servicios como SheetDB permiten lograr una conexion similar sin escribir Apps Script, ofreciendo un endpoint listo a cambio de conectar tu hoja de calculo desde su plataforma. Sin embargo, programar el script directamente otorga mayor control sobre la logica de procesamiento, validacion de datos y estructura de las columnas.
⚠
Consideracion de publicacion: Para que tu formulario funcione en linea (no solo localmente), debes subir tu archivo HTML a un repositorio de GitHub y activar GitHub Pages desde la configuracion del repositorio. Asi obtienes una URL publica donde cualquier usuario puede completar el formulario.
Verifica tu comprension: Cual es la principal limitacion del metodo mailto: para enviar formularios?

Aprendizaje Basado en Problemas — Entregable Integrador de Unidad

Proyecto Final  |  ABP

Formulario de Registro para un Evento Escolar

Disenaras, maquetaras y publicaras en linea un formulario de registro real para un evento escolar de tu eleccion, conectado a una hoja de Google Sheets que se actualiza automaticamente con cada registro recibido. Este proyecto integra cada uno de los cuatro modulos trabajados en la unidad.

Herramienta: Bloc de Notas + GitHub + Google Sheets Formato: Sitio web publicado (GitHub Pages) Trabajo: Individual o equipos de 2 Nivel: Licenciatura
P
Planteamiento del Problema

Tu universidad o un area estudiantil necesita digitalizar el registro de un evento escolar (un taller, una conferencia, una jornada cultural, un torneo deportivo, una asamblea estudiantil). Actualmente el registro se realiza en papel o mediante hojas de calculo compartidas manualmente, lo cual genera errores, perdida de informacion y falta de visibilidad en tiempo real para los organizadores.

Eventos sugeridos (el alumno elige uno aplicable a su contexto universitario):

  • Registro para un taller o curso extracurricular de tu facultad
  • Inscripcion a un club o sociedad de alumnos
  • Registro de asistencia a una conferencia o ponencia academica
  • Inscripcion a un torneo deportivo intramuros
  • Encuesta de interes para una jornada cultural o exposicion estudiantil
1
Investigacion y Planeacion 20 min

Antes de construir, define con claridad el alcance de tu formulario:

  • Selecciona el evento escolar y redacta una breve descripcion (2-3 lineas) de su proposito.
  • Lista la informacion minima necesaria a capturar: nombre, correo, carrera, turno o grupo, intereses o areas, comentarios.
  • Identifica que campos seran de texto, cuales de seleccion (radio, checkbox, select), y cuales requieren validacion especifica (correo, fecha, numero).
  • Define la paleta cromatica y tipografica de tu formulario, coherente con la identidad visual del evento.
2
Construccion del Formulario en HTML 50 min

Retoma como base tu archivo ejercicio-integral.html desarrollado a lo largo de la unidad, o construye una version nueva integrando obligatoriamente:

  • Estructura base: <form> con action y method="POST" correctamente configurados.
  • Campos de texto: minimo 2 campos <input type="text"> con sus respectivos <label> vinculados via for/id.
  • Campo de texto extenso: al menos 1 <textarea> para comentarios o motivo de registro.
  • Seleccion unica: un grupo de <input type="radio"> (minimo 2 opciones).
  • Seleccion multiple: un grupo de <input type="checkbox"> (minimo 3 opciones).
  • Menu desplegable: un <select> con minimo 4 <option>.
  • Validacion nativa: al menos un campo type="email", uno type="date" o type="number" con min/max, y minimo 3 campos marcados como required.
  • Usabilidad: uso de placeholder en al menos 2 campos.
  • Botones de accion: <button type="submit"> con texto claro y orientado a la accion (ej. "Confirmar mi registro").
3
Conexion con Google Sheets 30 min

Implementa el flujo de datos en vivo siguiendo el procedimiento del Modulo 4:

  • Crea una hoja de calculo en Google Sheets con encabezados que correspondan exactamente a los name de tus campos.
  • Configura el script de Google Apps Script (doPost) y publicalo como aplicacion web con acceso para "Cualquier usuario".
  • Sustituye el action de tu formulario con la URL del Webhook generado.
  • Realiza al menos 2 pruebas de envio y confirma visualmente que las filas aparezcan correctamente en la hoja de calculo.
4
Publicacion en GitHub Pages 15 min

Pon tu sitio en vivo para que sea accesible mediante una URL publica:

  • Crea un repositorio publico en GitHub (ej. registro-evento-[tu-apellido]).
  • Sube tu archivo HTML nombrandolo index.html.
  • Activa GitHub Pages desde Settings > Pages, seleccionando la rama principal como fuente.
  • Copia la URL publica generada y verifica que el formulario funcione correctamente desde otro dispositivo o navegador.
5
Documentacion y Reflexion Critica Colaborativa 20 min

Como parte del entregable, intercambia tu URL publicada con un companero de clase (construccion colaborativa del conocimiento) y completa lo siguiente:

  • Llena el formulario de tu companero como si fueras un usuario real del evento y observa el resultado en su hoja de Google Sheets (si te comparte acceso de visualizacion).
  • Redacta una retroalimentacion breve sobre la claridad, accesibilidad y diseno visual del formulario de tu companero.
  • En tu propio archivo, agrega una seccion final con un parrafo de reflexion que responda: que decisiones de diseno tomaste para tu publico objetivo (estudiantes universitarios), que dificultad tecnica enfrentaste y como la resolviste, y de que manera este formulario podria implementarse de forma real en tu universidad.
